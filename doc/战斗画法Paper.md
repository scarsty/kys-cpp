## 战斗画法 Paper

Paper 是一种带透视的纸片 3D 战斗画法，不再是独立战斗模式。它由 `game.battle_presentation == 1` 打开，当前可叠加到 `BattleSceneHades` 或 `BattleSceneSekiro` 的即时战斗逻辑上。战斗规则、角色移动、攻击特效、击退和 AI 仍分别由所选即时战斗模式负责，Paper 只替换战场的绘制、相机与输入映射。

画法目前仍是实验性质，重点用于验证原有 2D 战斗素材能否放入一个真 3D 场景中：地面铺在 `z = 0` 平面，建筑、人物和特效作为面向镜头的纸片竖立在地面上。

### 坐标和地面

Paper 画法内部使用 90 度物理坐标。普通战斗地图格子会通过 `pos45To90()` 转换到地面坐标：

- 地面位于 `z = 0`。
- 角色、建筑和特效使用 `{ x, y, z }` 作为世界坐标。
- 原本战斗图的 `y / 2` 伪透视不再作为 3D 坐标条件使用。

地面会先按 `smap` 地块生成一张原始尺寸的方形底图；如果存在当前战场的 `battle-earth` 完整地图，则再叠加这张预先绘制好的完整图。完整图中的透明缺块会露出下方 `smap` 底图来补齐。

`smap` 本身是半高等距素材。为了减少逐块纵向放大造成的接缝，当前实现会先按原版半高尺寸拼成完整的 `earth_base`，再把整张 `earth_base` 一次性纵向拉伸为 Paper 使用的方形 `earth` 地面纹理。拼接 `smap` 时按原始 tile 尺寸正常绘制，不额外做边缘扩绘。`earth_layer_` 中贴图索引为 0 的格子不会直接绘制原始贴图，会改用最近有效 `smap` 地块作为底图兜底；普通非 0 贴图里的黑色像素不会被抠除，也不会做纯黑透明处理。

为了避免半高等距地面在方形纹理四角留下透明区域，Paper 画法会在生成 `earth_base` 时扩大 `smap` 绘制循环，但只填满原始方形 render target 的范围，不再生成约 3 x 3 的超大扩展地面。范围外的虚拟格子会映射到最近的有效边界地块或最近边缘地块；这只影响底图填充，不改变碰撞和可行走范围。这个方案比大范围扩展更适合 Android，避免超大 render target 被硬件最大纹理尺寸裁剪。

绘制时地面会按纹理尺寸生成共享顶点网格，再用 `SDL_RenderGeometry` 提交，减轻单个大四边形的仿射贴图变形，同时避免逐块提交造成的接缝。当前网格固定为中间密、外圈疏：原始战场中央约 64% 的区域分配多数顶点，原始战场边缘使用中等密度。地面网格顶点带透明度：最外缘会淡出到完全透明，中心外侧有径向淡出，远处接近地平线的位置也会与天空做过渡。

### 镜头

Paper 画法使用 `Camera` 做透视投影。镜头每帧动态设置：

- 焦点优先为主角最近的敌人，`z = 0`。
- 若没有敌人，则使用场景中心。
- 镜头位置在主角身后约 400 像素，高度约 200 像素。
- 镜头会看向当前焦点。

这里的“身后”按主角与焦点的连线计算，因此相机方向会随主角和敌人的相对位置变化。

### 天空背景

Paper 画法不再加载外部天空贴图，而是在运行时绘制程序化天空背景。每次进入战斗会随机选择一种统一色调的天空模式：白天、黄昏、黑夜或黎明。天空由纵向渐变、太阳或月亮、夜间星点组成；太阳带柔和光晕，月亮是边缘较锐利的圆盘。绘制 3D 地面前，会根据当前相机的俯仰角计算地平线位置，并按相机水平朝向决定太阳、月亮和星点在画面中的位置。

云层使用 `resource/cloud` 贴图组运行时随机生成。每次进入战斗会生成一批云，并尽量遍历使用已有云贴图；每朵云都有随机角度、深度、位置、大小、透明度、视差和漂移速度。云的位置会受相机 yaw、相机 pitch 和时间漂移共同影响，即使视角静止也会缓慢移动。

天空和云只是背景层，不参与碰撞、遮挡和深度排序。云贴图不存在时只会少画云层，天空本身仍由程序化背景绘制。

### 纸片绘制

建筑、人物和攻击特效都按纸片方式绘制。每张纸片根据当前相机方向计算屏幕右轴 `paper_right`，再用贴图的 `dx/dy` 作为锚点展开成一个竖直四边形。

人物和特效的锚点来自角色或特效的世界坐标。建筑锚点来自战场建筑层格子转换后的地面坐标。空血或死亡角色会横着画，旋转方向参考角色当前面向。Paper 会在角色头顶的投影位置绘制血条；只狼模式还会显示架势条。

### 建筑和墙体

建筑层中部分编号表示墙体。这些墙体素材本身是分块拼接的，在 Paper 画法下若把每块直接画成独立纸片，无法有效拼合成连续墙面，反而容易出现断裂、错位和奇怪遮挡。因此墙体 tile 不会作为普通建筑纸片绘制，而是先把连续墙体区域合并为外轮廓边，再把轮廓边画成竖直墙面。当前墙体范围为：

- `701 ~ 1139`
- `1410 ~ 1436`
- `1505 ~ 1621`
- `1816 ~ 1849`
- `2116 ~ 2144`
- `2184 ~ 2285`

墙面贴图优先来自 `resource/paper-wall-texture/` 目录。该目录中的图片按原墙体编号命名，例如 `701.png`、`1410.png`。若不存在或加载失败，则回退到程序生成的 `paper-wall-edge` 简易条纹纹理。

`paper-wall-texture` 中的图片由 `tools/rectify_wall_textures.py` 从原始 `resource/smap/` 墙体素材转换得到。

墙体继续由即时战斗模式原有的 `isBuilding()`、`canWalk45()` / `canWalk90()` 与 AI 寻路处理；Paper 只改变其视觉表现。

### 绘制顺序

天空背景先绘制，随后绘制地面。建筑、人物和攻击特效会进入同一个纸片队列，按当前相机方向计算 depth，然后从远到近排序绘制。这样近处纸片会覆盖远处纸片，建筑和角色之间的遮挡关系会随镜头变化。

当前排序仍是 painter 算法，不是逐像素 Z-buffer。大建筑只用一个锚点计算深度，因此在相机角度变化或建筑较大时，仍可能出现整张纸片遮挡过强、局部遮挡不准确等问题。

### 输入和移动

主角移动使用相机相对方向：

- `W`：朝镜头前方，也就是焦点方向移动。
- `S`：朝镜头后方移动。
- `A`：朝画面左侧移动。
- `D`：朝画面右侧移动。
- 手柄左摇杆使用同一套相机相对方向。

方向键改为控制自由镜头；移动仍由所选即时战斗模式的碰撞、建筑、水面和角色逻辑约束。

### 当前限制

- `SDL_RenderGeometry` 不提供真正的透视正确 UV 插值，地面只能通过细分网格近似。
- 纸片排序是对象级 painter 算法，不是硬件深度缓冲。
- 建筑只用锚点参与排序，较大的建筑可能遮挡不够精确。
- 天空背景只是 2D 程序化背景近似，不是六面体或球形天空盒。
- 所有建筑、人物和特效都面向镜头，尚未区分更复杂的朝向和体积。
- 镜头跟随最近敌人和主角位置，快速移动或敌人切换时可能仍需要平滑处理。
