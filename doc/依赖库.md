### 主程序依赖

以下库是当前主程序直接使用的依赖。Windows 下建议用 vcpkg 安装；Linux 下优先使用系统包管理器；macOS 下可用 Homebrew。Windows 的完整安装命令见根目录 [readme.md](../readme.md)。

- SDL3 <https://www.libsdl.org/>：窗口、渲染、输入和触控。
  - SDL3_ttf <https://github.com/libsdl-org/SDL_ttf>：字体渲染。
  - SDL3_mixer <https://github.com/libsdl-org/SDL_mixer>：音乐和音效。
  - SDL3_image <https://github.com/libsdl-org/SDL_image>：PNG、WebP 等图片载入；高清资源需要 WebP 支持。
- libiconv <https://www.gnu.org/software/libiconv/>：文本编码转换。
- Lua <https://www.lua.org/>：兼容原有 Lua 剧本。
- SQLite3 <https://www.sqlite.org/>：新存档数据库。
- yaml-cpp <https://github.com/jbeder/yaml-cpp>：从 YAML 描述创建 RunNode/UI。
- Asio <https://think-async.com/Asio/>：联机战斗；仅在定义 `WITH_NETWORK` 时启用，属于仅头文件使用。
- PicoSHA2 <https://github.com/okdshin/PicoSHA2>：联机校验和，仅头文件使用。
- Glaze <https://github.com/stephenberry/glaze>：新存档的编译期反射与序列化辅助，仅头文件使用。

### 随仓库提供的组件

- hanz2piny <https://github.com/yangyangwithgnu/hanz2piny>：汉字转拼音实现，源码位于 `others/`。
- mlcc <https://github.com/scarsty/mlcc>：通用功能集合，作为子模块使用。
- cifa <https://github.com/scarsty/cifa>：C 风格脚本语言，作为子模块使用，可替代部分 Lua 脚本。
- OpenCC <https://github.com/BYVoid/OpenCC>：当前只使用其词库；主程序不直接调用 OpenCC 库。
- ZIP：主程序使用 mlcc 的 `ZipFile2`，其 STORE/DEFLATE 读写、CRC-32 和压缩实现均内置，不依赖 libzip 或 kuba--/zip。

### 视频

参见：<https://github.com/scarsty/smallpot>

smallpot 是可选的 Windows 视频播放器动态库，用于播放过场视频。`Video.cpp` 只会在 Windows 且定义 `WITH_SMALLPOT` 时调用它；Android、macOS 和 Linux 的常规构建不依赖该库。

如不需要视频播放，可移除项目中的 `WITH_SMALLPOT` 定义和 `smallpot.lib` 链接项。

如果仅需播放视频，建议用lsl330改造的lite版：<https://github.com/lsl330/smallpot-lite>。

### 间接使用的库

以下通常由 SDL3_image、SDL3_ttf、SDL3_mixer 或 smallpot 的包管理依赖自动带入，无需为主程序单独配置。FFmpeg 及其后的播放器相关库仅在启用 smallpot 时需要。

- zlib <https://zlib.net/>
- freetype <https://www.freetype.org/>
- libpng <http://www.libpng.org/pub/png/libpng.html>
- FFmpeg <https://www.ffmpeg.org/>
- libass <https://github.com/libass/libass>
- fribidi <https://www.fribidi.org/>
- harfbuzz <https://github.com/harfbuzz/harfbuzz>
- fontconfig <https://www.freedesktop.org/wiki/Software/fontconfig/>
- mpg123 <https://mpg123.org/>
- FluidSynth <https://www.fluidsynth.org/>

### 音频历史

早期 SDL_mixer 在本项目中曾出现严重的切换时崩溃问题，因此一度改用 BASS 和 BASSMIDI。当前 SDL3_mixer 的播放表现已满足需求，主程序已经切回开源的 SDL3_mixer；BASS 仅作为历史遗留记录，不再是当前依赖。

### 历史遗留配置

主程序已不再使用下列库；Windows `kys.vcxproj` 中仍保留 OpenCV include 路径等历史配置，不代表新的构建或运行需要它们。

- libzip <https://github.com/nih-at/libzip/>
- zip <https://github.com/kuba--/zip/>
- BASS, BASSMIDI <http://www.un4seen.com/> （闭源，但可以免费用于非商业项目）
- OpenCV：仅部分独立工具和 mlcc 的可选绘图辅助使用，主游戏不依赖。
