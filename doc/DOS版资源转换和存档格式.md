# DOS版资源转换和存档格式

在 `tools` 文件夹下面，有 3 个工具用于将 DOS 版（或 kys-pascal 相关）的资源转换的例程。请参阅该目录下面的文档。

了解了文件格式之后，可以让 AI 来编写这些脚本，稳定性甚至更高。

## 存档格式

存档的基础数据部分（R 部分）使用 SQLite 数据库保存。默认启用 `zip_save` 时，每个存档槽保存为 `save/<编号>.zip`：其中包含 `1.db`，以及未启用数据库 SD 数据时的 `s1.grp`、`d1.grp`。关闭 `zip_save` 时，R、S、D 分别保存为 `<编号>.db`、`allsin.grp`/`s<编号>.grp`、`alldef.grp`/`d<编号>.grp`。

读取存档时会优先读取 ZIP；若 ZIP 不存在，仍可回退到未压缩的 R/S/D 文件，并在启用 `zip_save` 时转换为 ZIP 存档。S、D 数据也可选择写入 SQLite 的 BLOB 表，但该选项默认关闭。

之前游戏使用的列表文件只保留了升级经验列表和离队列表，改用 txt 格式。

DOS 版初始存档中的中文字段可能仍是 CP950（BIG5）。abc 的扩展转换会将这些字符串转为 UTF-8，并导出 SQLite 数据库；`BaseInfo::Encode` 字段会保留在数据库中作为兼容数据。当前运行时的 `Save` 路径不根据该字段做文本编码转换，因此不要依赖“保存时自动转换编码”的旧行为。

## abc 工程

abc 工程用于转换旧数据。建议自行调整代码后，使用调试模式执行。它主要将 DOS 版的 16 位 GRP/IDX 存档扩展为 32 位数据，转换 CP950 字符串，并将 R 部分导出为 SQLite 数据库。存档的 S 和 D 部分仍使用 16 位整数，其余还有一些其他文件也没有进行转换。

以上提到的数据，除文本文件外均可以用新版 UpEdit 修改。
