# 高清素材的方案

## 主地图

游戏中需要将碎图拼成一张完整的图片。如使用算法或深度学习的模型将素材放大，直接拼接的话会出现接缝。因此，应先拼接成一张完整的地面图片（UpEdit有此功能），再将其放大。之后在游戏中再往上画建筑等元素。

因为主地图的尺寸较大，已经超过显卡的纹理尺寸，因此应使用程序拆散为一系列图片。此处是8*8的方案，生成64张图片。可以手动删除一些没有内容的图案片以节省空间。

一个范例程序如下：

```c++
int main()
{
    cv::Mat m = cv::imread("map.bmp");
    autocrop(m);
    size_t size = 8;
    int width = m.cols / size;
    int height = m.rows / size;
    int k = 0;
    for (int i = 0; i < size; i++)
    {
        for (int j = 0; j < size; j++)
        {
            cv::Mat m1 = m(cv::Rect(j * width, i * height, width, height));
            double minv, maxv;
            cv::minMaxLoc(m1, &minv, &maxv);
            if (minv != maxv)
            {
                cv::imwrite(std::format("{}.png", k), m1);
                std::print("{}: {},{}\n", k, minv, maxv);
            }
            k++;
        }
    }
    return 0;
}
```

之后将生成的图片放入mmap-earth，即可以画出地面。但是这种方式不支持地面的图片闪烁。但你可以用其他方法增加动态。

其他素材都是独立的图片，直接放大即可。

## 内景地图和战斗地图

预先拼好地面再放大放置于smap-earth和battle-earth，素材再单独放大。但是有一些墙体和占用多张图的物体仍然有接缝，这个问题暂时不好解决。

## 其他素材

光影和战斗图单独放大即可。

## 偏移的设置

因为放大后图片的尺寸发生了变化，所以需要重新设置偏移。一般来说直接乘以放大倍数即可。可以编写脚本实现。

## 程序中的修改

TILE_W和TILE_H的值需要修改为放大后的尺寸。其他还有地面高度，装饰高度等。
