# 棋池配置说明

## 概述

`config/chess_pool.yaml` 定义了自走棋商店中可出现的所有棋子及其费用等级。修改后重启游戏即可生效，无需重新编译。

## 文件格式

```yaml
- 费用: 1
  角色:
    - 130  # 柯鎮惡
    - 63   # 程英
- 费用: 2
  角色:
    - 54   # 袁承志
# ...共5个费用等级
```

- `费用`：1~5，对应棋子的购买价格和强度等级
- `角色`：该费用下所有角色的ID列表
- 注释中的名字仅供参考，游戏按ID读取角色数据

## 如何修改

- 添加棋子：在对应费用的 `角色` 列表中加入角色ID
- 移除棋子：删除对应行
- 调整费用：将角色ID移到另一个费用等级下

## 查询角色ID

角色ID存储在 `game/save/game.db` 的 `role` 表中。以下是几种查询方式：

### 方式一：SQLite 图形工具（推荐新手使用）

1. 下载 [DB Browser for SQLite](https://sqlitebrowser.org/)（免费开源）
2. 打开 `game/save/game.db`
3. 切换到「浏览数据」标签页，选择 `role` 表
4. 可直接搜索角色名，`编号` 列即为角色ID

### 方式二：命令行 sqlite3

```bash
sqlite3 game/save/game.db "SELECT 编号, 名字 FROM role WHERE 名字 LIKE '%郭靖%';"
```

### 方式三：Python 脚本

```python
import sqlite3
conn = sqlite3.connect('game/save/game.db')
cur = conn.cursor()
cur.execute("SELECT 编号, 名字 FROM role WHERE 名字 LIKE '%郭靖%'")
print(cur.fetchall())
```

## 平衡设计

### 费用等级 = 强度等级

棋池的核心原则：**费用等级直接决定棋子的基础强度**。每个棋子的属性被标准化到其费用对应的区间：

| 费用 | 等级 | 生命 | 攻击 | 速度 | 防御 |
|------|------|------|------|------|------|
| 1费 | 5~8 | 450~600 | 48~65 | 30~55 | 35~55 |
| 2费 | 10~12 | 551~684 | 68~88 | 40~65 | 50~70 |
| 3费 | 15~17 | 616~774 | 95~115 | 52~80 | 68~90 |
| 4费 | 20~23 | 721~861 | 120~145 | 68~100 | 85~120 |
| 5费 | 25 | 819~935 | 155~180 | 85~120 | 110~150 |

### 同费用内的差异化

同一费用等级内，棋子通过属性分配和技能组合形成不同定位：
- **高攻低防**（玻璃大炮）：如血刀老祖(2费, 攻击88, 防御52)
- **高防低攻**（坦克）：如枯荣(2费, 攻击68, 防御70)
- **高速**（刺客）：如云中鹤(2费, 速度65)
- **均衡型**：如黄蓉(2费, 各项中等偏上)

### 技能的作用

每个棋子配备两个技能，这是同费用棋子之间最大的差异来源：
- 低费棋子配基础武功（如罗汉拳、松风剑法）
- 高费棋子配强力绝学（如降龙十八掌、六脉神剑）
- 部分技能的威力曲线经过额外调整，确保伤害与费用等级匹配

### 星级缩放

棋子升星后按 `chess_balance.yaml` 中的星级加成公式缩放。加成同时包含百分比和固定值：
- 低费棋子从固定值中获益更大（相对于基础属性占比高）
- 高费棋子从百分比中获益更大（基数大）
- 这使得3星1费棋子在中期仍有竞争力，而非被高费棋子完全碾压

### 添加新棋子

1. 在 `chess_pool.yaml` 中将角色ID加入对应费用等级
2. 用 SQLite 工具打开 `game/save/game.db`，修改该角色在 `role` 表中的属性，参考上方同费用的数值区间
3. 如需加入羁绊，编辑 `config/chess_combo.yaml`
