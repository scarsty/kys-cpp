# 平衡配置说明

本文档介绍如何通过修改配置文件来调整自走棋的数值平衡。

## 配置文件位置

```
game/config/chess_balance.yaml
```

游戏启动后会自动读取此文件。如果文件格式有误，控制台会输出错误信息。

## 星级加成

控制棋子升星后的属性倍率。每颗星按基础属性乘以对应倍率。

```yaml
星级加成:
  生命攻击倍率: 0.80    # 每星+80%生命和攻击
  防御倍率: 0.50        # 每星+50%防御和武学熟练度
  速度倍率: 0.25        # 每星+25%速度
```

例如1星棋子的生命 = 基础生命 × (1 + 0.80) = 1.8倍。

## 经济

控制金币收入和花费。

```yaml
经济:
  初始金币: 20          # 开局金币
  刷新费用: 2           # 刷新商店花费
  购买经验费用: 5       # 购买经验花费
  购买经验数量: 5       # 每次购买获得的经验
  战斗经验: 4           # 每次胜利自动获得的经验
  战斗奖励基础: 3       # 胜利奖励基础值
  战斗奖励阶段系数: 2   # 每阶段额外奖励
```

战斗奖励公式：`基础 + 阶段 × 系数 + 当前关卡序号`

## 棋子费用

```yaml
棋子费用:
  各费价格: [1, 2, 3, 4, 5]   # 1~5费棋子的购买价格
  星级倍率: 3                   # 出售时星级价格倍率
```

出售价格 = 费用 × 倍率^星级 × 数量

## 经验与等级

```yaml
经验表: [4, 6, 10, 14, 18, 22, 26, 32, 38, 50]
最高等级: 10
```

经验表第N项为从N级升到N+1级所需的经验。升级后经验清零重新累积。等级决定出战人数上限和商店刷新品质。

## 容量

```yaml
背包上限: 10       # 备战席最大棋子数
最低出战人数: 2    # 出战人数下限（等级低于此值时使用）
```

## 商店权重

每行对应一个等级（0~9级），5个数字分别是1~5费棋子的出现权重。

```yaml
商店权重:
  - [100, 0, 0, 0, 0]      # 0级：只出1费
  - [70, 30, 0, 0, 0]       # 1级
  - [60, 35, 5, 0, 0]       # 2级
  # ...以此类推
```

权重之和应为100。

## 敌人

控制敌方棋子的生成难度。

```yaml
敌人:
  数量基础: 2              # 敌人基础数量
  数量上限: 10             # 敌人最大数量
  低级混入概率: 30         # 普通战低一费敌人出现概率(%)
  星级提升起始关卡: 2      # 从第几关开始可能出现升星敌人
  星级提升概率: 20         # 升星概率(%)
  Boss升阶: 1              # Boss比当前阶段高几费
  Boss最高费: 4            # Boss最高费用等级(0~4)
  Boss星级起始阶段: 4      # 从第几阶段开始Boss有星级
```

敌人数量公式：`min(上限, 基础 + 阶段 + 关卡序号)`

## 进度

```yaml
进度:
  每阶段关卡数: 5    # 每个阶段包含几关
  Boss关卡序号: 4    # 第几关是Boss战（从0开始）
  通关阶段数: 3      # 通关需要完成几个阶段
```
