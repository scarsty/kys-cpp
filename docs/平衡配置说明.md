# 平衡配置说明

本文档介绍如何通过修改配置文件来调整自走棋的数值平衡。

## 配置文件

| 文件 | 用途 |
|------|------|
| `config/chess_balance.yaml` | 主平衡配置 |
| `config/chess_pool.yaml` | 棋池配置（角色ID与费用等级） |
| `config/chess_neigong.yaml` | 内功配置 |
| `config/chess_challenge.yaml` | 远征挑战配置 |

游戏启动后会自动读取这些文件。如果文件格式有误，控制台会输出错误信息。

## 星级加成

控制棋子升星后的属性加成。每颗额外星同时按百分比和固定值提升属性。

```yaml
星级加成:
  生命倍率: 0.4       # 每星+40%生命
  攻击倍率: 0.4       # 每星+40%攻击
  防御倍率: 0.4       # 每星+40%防御
  速度倍率: 0.25      # 每星+25%速度
  固定生命: 275       # 每星额外固定生命
  固定攻击: 55        # 每星额外固定攻击
  固定防御: 35        # 每星额外固定防御
```

例如1星棋子的生命 = 基础生命 × (1 + 0.4) + 275。

## 经济

控制金币收入和花费。

```yaml
经济:
  初始金币: 15
  刷新费用: 2           # 刷新商店花费
  购买经验费用: 5       # 购买经验花费
  购买经验数量: 5       # 每次购买获得的经验
  战斗经验: 2           # 每次战斗自动获得的经验
  Boss战斗经验: 6       # Boss战胜利获得的经验
  战斗奖励基础: 4       # 胜利奖励基础金币
  战斗奖励增长: 12      # 最后一关比第一关多给的金币(逐关递增)
  Boss奖励加成: 5       # Boss战额外奖励金币
  利息百分比: 10        # 每回合按存款给利息(百分比)
  利息上限: 3           # 利息最多给几金币
```

## 棋子费用

```yaml
棋子费用:
  各费价格: [1, 2, 3, 4, 5]   # 1~5费棋子的购买价格
  星级倍率: 3                   # 出售时星级价格倍率
```

出售价格 = 费用 × 倍率^星级 × 数量

## 经验与等级

```yaml
经验表: [1, 5, 10, 15, 20, 25, 30, 35, 40]
最高等级: 9
```

经验表第N项为从N级升到N+1级所需的经验。升级后经验清零重新累积。等级决定出战人数上限和商店刷新品质。

## 容量

```yaml
背包上限: 10       # 备战席最大棋子数
最低出战人数: 2    # 出战人数下限（等级低于此值时使用）
```

## 商店权重

每行对应一个等级（0~9级），5个数字分别是1~5费棋子的出现权重。

```yaml
商店权重:
  - [85, 15, 0, 0, 0]      # 0级
  - [70, 30, 0, 0, 0]       # 1级
  - [60, 35, 5, 0, 0]       # 2级
  # ...以此类推
```

权重之和应为100。

## 敌人表

每关敌人以列表形式直接配置，每个敌人为 `[费用tier, 星级star]`。

```yaml
敌人表:
  - [[1, 1], [1, 1]]                          # 第1关
  - [[1, 1], [2, 1]]                          # 第2关
  - [[1, 1], [2, 1]]                          # 第3关
  - [[1, 2], [2, 1], [2, 1]]                  # 第4关(BOSS)
  # ...以此类推
```

每隔 `Boss间隔` 关出现一次Boss战。敌人数量和强度完全由此表决定。

## 进度

```yaml
进度:
  总关卡数: 28       # 通关所需总关卡数
  Boss间隔: 4        # 每隔几关出现Boss
```

## 内功配置 (chess_neigong.yaml)

```yaml
刷新费用: 4          # 内功刷新花费
选择数量: 3          # 每次可选内功数量
Boss可选层级:        # boss序号(0-indexed) -> 可选tier列表
  0: [1]
  1: [1, 2]
  2: [1, 2, 3]
  3: [1, 2, 3, 4]
```

`层级分配` 将武功ID分配到不同层级，`效果` 定义每个武功的战斗效果。

## 远征挑战配置 (chess_challenge.yaml)

```yaml
远征挑战:
  - 名称: "聚贤庄内"
    描述: "挑战乔峰"
    敌人: [[50, 3]]           # [角色ID, 星级]
    奖励:
      - 类型: 获取金币
        数值: 10
      - 类型: 升星1到2
        最高费用: 2
      - 类型: 获取内功
        最高层级: 1
```

奖励类型：`获取金币`、`获取棋子`、`获取内功`、`升星1到2`、`升星2到3`。
